# Portfolio Anatomy

Portfolio sections, conversion patterns, and what NOT to include. Use during Phase 6 planning when `--type=portfolio`.

## Core Difference vs Landing

| Aspect | Landing | Portfolio |
|--------|---------|-----------|
| Conversion goal | Buy / signup / demo | Hire-me / book call / freelance inquiry |
| Visitor mindset | Evaluating product | Evaluating person + craft |
| Lead element | Product value statement | The work itself |
| Trust signal | Customer logos + testimonials | Project quality + client list |
| Volume of CTAs | 2-3 across page (same destination) | 1-2 (contact / availability) |
| Length | Long, multiple sections | Often shorter — work does the talking |

**Critical principle:** lead with WORK, not personality. The visitor wants to see what you've made before they care who you are.

## Standard Section Stack

A portfolio usually contains 5–7 of these sections.

| # | Section | Question it answers | Optional? |
|---|---------|---------------------|-----------|
| 1 | Hero (intro) | Who is this and what do they make? | NO |
| 2 | Selected Work Grid | What does their work actually look like? | NO |
| 3 | Featured Case Study | How do they think? | recommended |
| 4 | About / Bio | Who is the person behind this? | NO |
| 5 | Process / Approach | How do they work? | optional |
| 6 | Selected Press / Awards | Has anyone validated this? | optional |
| 7 | Contact / Availability CTA | How do I reach them? | NO |
| 8 | Footer | Where else? | NO |

Avoid: skill bars / percentages, "I am proficient in..." tag clouds, "Years of experience" counter, downloadable resume button on top of the fold.

## Section-by-Section Patterns

### 1. Hero (Intro)

**Goal:** In 5 seconds, the visitor knows what you make and whether to keep scrolling.

**Required elements:**
- Owner one-liner — what you do, plainly stated (NOT cute, NOT clever)
- Optional: location / availability status ("Available for freelance from June 2026")
- Optional: featured project sneak-peek as visual anchor

**Layout patterns by vibe:**
| Vibe | Hero layout |
|------|------------|
| Minimal | Large display name + 1-line craft, generous whitespace, single visual |
| Editorial | Asymmetric split with display "Your Name" + 1-line + featured-work peek |
| Brutalist | Full-bleed featured-work image with overlaid raw type |
| Hand-crafted | Hand-drawn elements, name as signature, bio in serif |
| Luxury | Symmetrical centered, refined, single project crop |

**Anti-patterns (refuse these):**
- "Hi, I'm [Name], a passionate designer who loves coffee" — cliché opener
- "Hello, world!" coding cliché
- "Welcome to my portfolio" — generic
- Long bio paragraph above the work — visitor leaves
- Selfie photo as primary hero element — work is the hero, not the face
- "Multi-disciplinary creative based in [city]" — vague, every portfolio says this
- Animated typewriter intro — dated

**Approved patterns:**
- "Sarah Chen — Brand identity for early-stage tech." (specific craft + audience)
- "I design healthcare apps. 6 years, 3 platforms, 12 launches." (concrete proof)
- Featured project crop above the fold, name + craft below
- "Available from {date}" status indicator

### 2. Selected Work Grid

**Goal:** Show 4-12 best projects. Quality > quantity.

**Patterns by vibe + work type:**

| Pattern | Best for |
|---------|----------|
| Bento grid (mixed sizes) | Visual designers, illustrators — composition matters |
| Uniform 2-col / 3-col grid | Product designers — consistency conveys process |
| Single full-width per project (scrolling reveal) | Architects, photographers — work needs space |
| Masonry (variable heights) | Editorial, hand-crafted vibes |
| List view (text + thumbnail) | Writers, strategists — process-led work |

**Each project tile contains:**
- Cover image (real project work, NOT a mockup of a phone holding the work)
- Project title (specific — "Healthtech onboarding flow" NOT "Project 1")
- Client name (if shareable)
- 1-3 tags (Brand / Web / Product / Identity / etc.)
- Year (organic ordering signal)
- Optional: 1-line context

**Anti-patterns:**
- "Project 01" / "Project 02" titles — generic
- All projects same aspect ratio forced by CSS — flatten composition
- iPhone-mockup-holding-the-work cliché — show the work directly
- 30+ projects on grid — dilution; cut ruthlessly to 6-12
- Hover-effect overload (zoom + tilt + glow + caption-slide) — let work speak
- Filter buttons by category at top of grid — usually unused, adds complexity
- Grayscale-until-hover treatment — gimmick, hides the actual color of the work

**Approved patterns:**
- Real project covers at original aspect ratios (not forced uniform)
- Calm grid with subtle on-hover state (small scale, color shift, or just title appearing)
- Year + client visible at rest — date the work
- Click → either project page (if case studies) or external link (live work)

### 3. Featured Case Study (1-2)

**Goal:** Show how you think. ONE or TWO deep dives, not all projects deep.

**Structure (per case study):**
- Title + client + role + duration + year
- Context (the problem in 2-3 sentences)
- Your contribution (specific — "led brand identity, art-directed photography, designed packaging system")
- Process snapshots (3-6 images / sketches / iterations)
- Outcome (specific result if shareable, or qualitative observation)
- Final hero image / screenshot
- Link to live work if applicable

**Patterns:**
- Long-form scroll case study (own page)
- Inline case study (collapsible on portfolio main)
- Magazine spread layout

**Anti-patterns:**
- Case study without client/context — looks fictional
- "Final design" only, no process — looks like you got it right first try
- Generic process labels ("Discover / Define / Develop / Deliver") — every agency says this
- Hiding the actual final work behind too many process shots

### 4. About / Bio

**Goal:** Reduce hiring-anxiety. Visitor wants to know: real human, real experience, low risk.

**Required elements:**
- Real bio in your voice — not third-person agency-speak
- Career path in 2-3 sentences (where you've worked, what you focus on)
- Current location / timezone (matters for remote work)
- Availability status (open / booked / waitlist)

**Optional:**
- Selected client list (companies you've worked with)
- Awards / press mentions
- Speaking / teaching engagements
- Personal interests (only if relevant to work — NOT "I love coffee and dogs")

**Anti-patterns:**
- "Multi-disciplinary creative passionate about pixel-perfect design" — ChatGPT bio
- Listing every tool you know ("Figma, Sketch, Photoshop, Illustrator, After Effects...") — tools don't matter, work does
- Skill bars showing "Figma 90% / HTML 75%" — childish, removes trust
- "Years of experience" prominent counter — usually undermines
- Long childhood-to-now narrative — boring; lead with current focus

**Approved patterns:**
- "I've spent the last 4 years on healthcare product design — currently at Oscar, previously at One Medical."
- Selected client list as logo bar OR text list — concrete proof
- "Currently booking projects starting June 2026" — actionable

### 5. Process / Approach (optional)

**Goal:** For service-style portfolios (freelance, agency-style). How you engage with clients.

**Structure:**
- 3-5 step explanation of how you work
- Each step: short title + 1-2 sentence body
- Optional: typical timeline + investment range

**Anti-patterns:**
- "Discover → Define → Design → Deliver" 4D framework — every agency
- Steps with stock illustrations — generic
- Process diagram with circular arrows — vague

**Approved patterns:**
- Specific named phases tied to your craft
- Timeline ranges ("3-6 weeks for brand identity")
- Investment guidance ("$8k–$25k depending on scope")
- Single sentence describing what makes your process distinct

### 6. Selected Press / Awards (optional)

**Goal:** Borrow third-party validation.

**Patterns:**
- Logo bar (publication + award logos)
- "Featured in [Publication], [Year]" linked text list
- Quote pulls from press articles

**Anti-patterns:**
- 30+ logos — desperate
- Generic "Featured in" without sources — meaningless
- Awards from your own family member's blog — undercuts trust

### 7. Contact / Availability CTA

**Goal:** Make hiring you frictionless.

**Required:**
- Email link (real, monospace if vibe matches)
- Availability status (open / booked / waitlist)
- 1-line invitation ("For project inquiries, write me at...")

**Optional:**
- Contact form (only if needed — email is usually fine)
- Calendly / scheduling link
- Social handles (X/Twitter, LinkedIn, Instagram)

**Anti-patterns:**
- "Let's create magic together!" — vague, AI-cliché
- "Drop a line" — overused
- "Hire me" button alone with no context — too direct
- Contact form with 8 required fields — friction
- "Looking for opportunities" without dates — passive

**Approved patterns:**
- "Available for new projects from {date}. Email {address}."
- "Selectively booking design partnerships through Q3."
- "For collaborations, write me at {email}."

### 8. Footer

**Goal:** Functional minimum.

**Required:**
- Year + name (copyright)
- Email link (repeat from contact section)
- 2-4 social links (custom icons from icon set, NOT library)

**Anti-patterns:**
- 6-column footer with site map — overkill for portfolio
- "Built with Next.js + Tailwind + Vercel" tech-stack flex — usually noise
- Newsletter form takeover — distracts from contact CTA above

## Conversion Heuristics for Portfolio

### The 5-Second Test (portfolio version)
Show the hero for 5 seconds, then ask:
1. What does this person make?
2. Is the work any good (visible above the fold)?
3. How do I get in touch?

If any unclear, hero fails.

### CTA Discipline
- Hero: optional secondary "View work" or jump anchor
- Mid-page: 0 CTAs (let work speak)
- Final section: 1 primary CTA = email contact

Total: max 2 CTAs. Both lead to email.

### Above-the-Fold Discipline
On 1366×768 desktop and iPhone 14 (390×844):
- Owner one-liner visible
- At least 1 piece of work visible (or work-as-hero)
- Availability indicator (if applicable) visible

If the visitor cannot see ANY of your actual work above the fold, restructure.

```

├── app/
│   ├── page.tsx                      # Composes all sections
│   ├── layout.tsx                    # Fonts, metadata
│   ├── globals.css                   # Tailwind + tokens
│   ├── work/
│   │   └── [slug]/

│   └── components/
│       ├── icons/                    # Custom SVG icon set
│       ├── three/                    # 3D (if applicable)
│       ├── sections/
│       │   ├── hero-intro.tsx
│       │   ├── work-grid.tsx
│       │   ├── featured-case-study.tsx
│       │   ├── about-bio.tsx
│       │   ├── process.tsx           # optional
│       │   ├── press.tsx             # optional
│       │   ├── contact-cta.tsx
│       │   └── footer.tsx
│       └── ui/                       # shadcn primitives

    ├── content.ts                    # Bio + section copy

```


## Section Removal Rules

If unsure whether to include, default to remove. A 5-section portfolio with 6 strong projects beats a 9-section portfolio with 12 weak ones.

Always remove:
- Skill bars / proficiency percentages
- Tool tag clouds
- Quote of the day / favorite quote
- Long childhood-to-now bio narrative
- Generic process diagrams ("Discover-Define-Develop-Deliver")
- Empty "Coming soon" project tiles

## Mobile Discipline

- Every section works at 390px viewport
- Work grid: bento → 1-col stack OR 2-col with smaller tiles
- Case study: long scroll, no parallax overload
- Hero: text-above OR text-below image stacking, never side-by-side under 768px

## Type-Specific 3D Guidance

If user opted-in to 3D in Phase 2d:
- **Best fit:** interactive accent (3D logo / mark in nav, draggable item)
- **Acceptable:** scroll-triggered reveal between work grid items
- **Risky:** full hero 3D scene — risks overshadowing the actual work; only proceed if vibe is retro-futuristic / glass-tech AND user explicitly approves

A portfolio's 3D should support the work, never compete with it.

## Final Anti-Slop Audit (portfolio-specific)

Before declaring complete, verify:

- [ ] No "Hi, I'm X, a passionate..." opener
- [ ] No "Hello, world!" greeting
- [ ] No "Welcome to my portfolio" header
- [ ] No skill bars / proficiency percentages
- [ ] No tool tag clouds
- [ ] No 4D process framework (Discover-Define-Develop-Deliver)
- [ ] No iPhone-mockup-holding-the-work in project tiles
- [ ] No "Years of experience" prominent counter
- [ ] No "Multi-disciplinary creative" bio phrase
- [ ] No 30+ projects in grid (cut to 6-12)
- [ ] Real project covers, not stylized mockups
- [ ] Email + availability visible in contact section
- [ ] Plus all generic anti-slop rules from `anti-slop-rules.md`

## Strategic Omissions — what AI typically forgets

Audit before ship. Each item tagged with applicability condition.

- Privacy policy + terms-of-service links in footer `[always]`
- Custom 404 page `[when multi-page site]` (portfolios with per-project subpages)
- Form validation (client-side, inline) `[when contact form]`
- "Skip to main content" a11y link `[always]`
- "Back" navigation in case-study subpages `[when case studies have own pages]`
- Page metadata (`<title>`, description, OG image, social cards) `[always]`
- Working contact mechanism that actually mailtos / submits — not a dead form `[always]`
- Honest copy — no fabricated work / clients / awards; see `anti-slop-rules.md § Honest Copy Mandate` `[always]`

Cookie consent + sitemap typically skipped for single-page portfolios; revisit if multi-page or analytics-heavy.
